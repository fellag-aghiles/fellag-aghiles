<details>
<summary><b>🔍 Why Semantic Search vs Keyword Search</b></summary>

<br/>

| Query | Keyword Search | Semantic Search |
|-------|---------------|-----------------|
| `"invoice"` | Returns docs with the exact word "invoice" | Returns docs about billing, payments, receipts |
| `"slow response"` | Returns docs with "slow response" literally | Returns docs about latency, performance, timeouts |
| `"how do I cancel"` | Returns docs with exact phrase | Returns cancellation policy, refund process, account deletion |
| `"Python async code"` | Exact match only | Returns asyncio, coroutines, await patterns |

**Result:** Semantic search finds what users *mean*, not just what they *typed.*
Built with sentence-transformers + Qdrant + PostgreSQL FTS + Reciprocal Rank Fusion.

</details>
